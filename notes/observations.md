## Сессия 2026-04-22 (до HSV-верификации, до перерегистрации)

Источник: скриншоты + записи пользователя.

### Производительность (proc time)
- 0 объектов: 150–200 ms
- 1 объект: 240–250 ms
- 2 объекта: 290–330 ms
- 4 объекта: 440–450 ms
- Рост линейный — каждый объект добавляет ~70–80 ms (FLANN knnMatch × N объектов)

### Условия съёмки
- Камера не строго вертикальна — держится на стопке книг + опоре, лёгкий наклон
- «Плоская» плоскость поля — под небольшим углом к объективу
- Использовалась брошь 1 см на краю фишки для наклона ~20–30° при регистрации; визуально кадр не менялся

### Стабильность детекции
- chip-50 (чёрная): bbox джиттерит даже в одиночку при смещении от точки регистрации
- chip-5 (красная): иногда не детектируется рядом с chip-50 на одной горизонтали; chip-50 "съедает" chip-5
- chip-25 (синяя): больше ложных срабатываний с chip-5 (два box'а на каждой)
- chip-10: зарегистрирована на другом месте, детектируется хорошо
- Все 4 фишки: ~90–100 признаков после регистрации (per_shot=100); зелёные боксы редко строго прямоугольные

## Сессия 2026-04-23 (после HSV-верификации + перерегистрации, per_shot=100)

Источник: скриншоты + записи пользователя.

### Производительность
- 3 фишки: 350–360 ms
- 4 фишки: 400–430 ms
- 5 фишек: 520–530 ms

### Результаты
- chip-5 (красная), chip-25 (синяя): превосходная детекция без изъянов при 1–5 зарегистрированных
- chip-10 (зелёная): нестабильна при ≥3, "восьмёрка" при 4 зарегистрированных
- chip-50 (чёрная, глянец): нестабильна, bbox нормальный когда виден, но пропадает часто

**Вывод: HSV-верификация + мультиснимок превосходно работают для матовых плоских разноцветных фишек нечёрного цвета.**

## Сессия 2026-04-24 (перерегистрация, per_shot=150, 8 объектов)

Источник: scanner.log + скриншоты.

### Условия съёмки
- Первые 4 фишки (chip-5, chip-25, chip-50, chip-1) — в нормальном освещении
- chip-10 и все последующие — в тени (телефон/плотный объект убирает блики от камеры)
- Тень улучшает качество захвата для блестящих объектов: меньше бликов → стабильнее ключевые точки

### Количество признаков (per_shot=150, 5 снимков)
| Объект | Признаков | Примечание |
|---|---|---|
| chip-10 | 750 | максимум (150×5) |
| chip-25 | 696 | |
| chip-5 | 680 | |
| chip-1 | 622 | белая фишка |
| chip-50 | 601 | чёрная с глянцем |
| vili | 335 | объёмная металлическая подставка |
| lil-blue | 266 | синий камень, мало текстуры |
| lil-white | 224 | USB-модуль, мало текстуры |

Закономерность: больше выраженной текстуры → больше признаков → лучше детекция.

### Производительность
- 8 зарегистрированных, 5 активных: ~574 ms proc time, ~1.7 fps
- Рост с ~70–80 ms/объект подтверждается

### Детекция по объектам

**chip-5 (красная):** Стабильна. Редкие кратковременные пропадания (≤10 с), затем восстанавливается.

**chip-25 (синяя):** Заметно хуже чем в сессии 2. Частые пропадания при 8 зарегистрированных. С меньшим числом объектов — хорошо.

**chip-50 (чёрная, глянец):** Много циклов ПОЯВИЛАСЬ/ПРОПАЛА в логе (~каждые 1–3 мин). Bbox нормальный когда виден. При регистрации снимок 4 занял 43 с — возможно, неудачное освещение.

**chip-1 (белая):** Редкие детекции. Bbox деформирован — почти прямая линия. Причины: 1) HSV-фильтр отклоняет (белый близок к белому фону/бликам), 2) низкий контраст на белом фоне.

**chip-10 (зелёная):** Редкие детекции. Bbox деформирован — почти прямая линия. Причина неясна: признаков максимум (750), цвет хорошо отличим. Возможно, регистрация в тени дала другой профиль, чем условия детекции (нормальное освещение).

**lil-blue (синий камень):** 266 признаков — ниже порога надёжной детекции. Нестабильна, частые пропадания. Под прямым углом иногда работает.

**vili (металлическая подставка):** 335 признаков. Нестабильна. Иногда проскакивает.

**lil-white (USB-модуль):** 224 признака. Почти не детектируется. Первая регистрация прервана (ESC, 33 признака, опечатка в имени), переделана полностью — всё равно 224.

### "Восьмёрка" и деформированный bbox
- chip-10, chip-50: периодическая "восьмёрка"
- chip-1: bbox в виде почти прямой линии — более крайняя деградация гомографии

## Общие наблюдения

- Объекты с ≤300 признаками (lil-blue, vili, lil-white) детектируются плохо или почти никак
- Белые объекты (chip-1): HSV-верификация может отклонять из-за близости к фону
- Чёрные объекты (chip-50): низкий V → нестабильная HSV-верификация, глянец добавляет нестабильность ключевых точек
- Регистрация в тени помогает для глянцевых объектов
- chip-5 и chip-25 — эталон стабильной детекции при текущей системе
