# Game Piece Scanner

Система компьютерного зрения для идентификации и трекинга игровых фишек на поле.
Стек: Python · OpenCV (SIFT/AKAZE, CLAHE, FLANN, Homography) · NumPy

---

## Требования

- Python 3.13
- Windows (протестировано на Windows 11)
- Камера: USB-вебкамера, DroidCam, или телефон через scrcpy + OBS Virtual Camera

---

## Установка

```bash
python -m venv .venv
.venv/Scripts/pip install -r requirements.txt
```

---

## Настройка камеры

### Вариант A — scrcpy + OBS Virtual Camera (рекомендуется)

1. Включи отладку по USB на телефоне, подключи по USB
2. Запусти scrcpy с камерой телефона:
   ```
   scrcpy --video-source=camera --camera-facing=back --no-audio
   ```
3. В OBS Studio: добавь источник **«Захват окна»** → выбери окно scrcpy
4. Нажми **«Запуск виртуальной камеры»** (панель Управление)
5. Найди индекс OBS-камеры:
   ```bash
   .venv/Scripts/python -c "
   import cv2
   for i in range(5):
       cap = cv2.VideoCapture(i)
       ret, frame = cap.read()
       if ret: print(i, int(cap.get(3)), 'x', int(cap.get(4)))
       cap.release()
   "
   ```
6. Укажи найденный индекс в `config/settings.py` → `CAMERA_USB_INDEX`

### Вариант B — DroidCam USB

1. Установи DroidCam на ПК и телефон, подключи по USB
2. В `config/settings.py`: раскомментируй строку с `cv2.CAP_DSHOW` в `camera_manager.py`
3. Укажи `CAMERA_USB_INDEX = 1` (обычно)

## Управление

| Клавиша / Кнопка | Действие |
|---|---|
| **R** | Зарегистрировать фишку (ввод имени → выделить мышью) |
| **L** | Список зарегистрированных фишек |
| **D** | Удалить фишку |
| **I** | Просмотр изображений зарегистрированных фишек |

| **ESC** | Отменить текущий ввод / регистрацию |

## Профилирование производительности

Профайлер замеряет время каждой операции внутри одного кадра и показывает, где уходит `proc time`.

**Включение:** `config/settings.py` → `PROFILING_ENABLED = True`

**Что замеряется:**

| Секция | Что включает |
|---|---|
| `camera` | Чтение кадра с камеры (`cap.read()`) |
| `extract` | SIFT/AKAZE на полном кадре — обычно самая тяжёлая операция |
| `detect` | FLANN knnMatch + ratio test + RANSAC/гомография для каждого объекта. Растёт линейно с числом зарегистрированных фишек |
| `tracker` | Обновление истории позиций, медианное сглаживание |
| `draw` | Отрисовка оверлея, кнопок, текста на кадре |

**Как пользоваться:**
1. Запусти систему с `PROFILING_ENABLED = True`
2. Дай поработать ~60 кадров (авто-вывод в консоль каждые 60 кадров)
3. Нажми **P** — сохранит базовую линию в `metrics/profile_baseline.csv`
4. Сравнивай с базовой линией после любых изменений настроек или алгоритма

**Что можно узнать:**
- Если `extract` > 200 ms → снизить `SIFT_FEATURES` или уменьшить разрешение
- Если `detect` растёт с каждой новой фишкой → снизить `FLANN_CHECKS` или `SIFT_FEATURES`  
- Если `camera` > 50 ms → узкое место в захвате (буфер, кодек, USB)
- Соотношение `extract` / `detect` показывает, что выгоднее оптимизировать первым

## Ключевые настройки (`config/settings.py`)

| Параметр | По умолчанию | Описание |
|---|---|---|
| `DETECTOR_TYPE` | `'sift'` | Детектор признаков: `'sift'` или `'akaze'` |
| `CAMERA_WIDTH/HEIGHT` | `1280 / 720` | Разрешение захвата. При смене — перерегистрировать фишки |
| `CAMERA_MJPG` | `True` | MJPG-поток: в 10–30× больше fps по USB vs raw YUY2 |
| `CAMERA_FPS` | `30` | Целевой fps (запрос к драйверу) |
| `DISPLAY_SCALE` | `0.75` | Масштаб окна; обработка всегда на полном разрешении |
| `MATCH_RATIO_THRESHOLD` | `0.80` | Порог ratio test; ниже → строже, меньше ложных срабатываний |
| `MIN_MATCHES_THRESHOLD` | `8` | Мин. совпадений для распознавания фишки |

### Fps/разрешение по USB

| Режим | 1280×720 | 1920×1080 |
|---|---|---|
| Raw YUY2 (`CAMERA_MJPG=False`) | ~5 fps | ~1 fps |
| MJPG (`CAMERA_MJPG=True`) | ~30 fps | ~10 fps |
| OBS Virtual Camera | 60 fps | 60 fps |

## Как работает детекция кадра (SIFT)

SIFT обрабатывает полный кадр (например, 1280×720) и находит до `SIFT_FEATURES` «интересных» точек — углов, перепадов яркости, краёв. Для каждой точки вычисляется 128-мерный дескриптор (гистограммы градиентов в окрестности 16×16 пикселей).

**Ключевое ограничение — бюджет точек.**  
`SIFT_FEATURES = 500` — это лимит на весь кадр. Эти 500 точек распределяются между фоном, всеми фишками и краями стола. С ростом числа объектов на каждую фишку приходится меньше точек:

| Объектов на поле | ~точек на фишку | Совпадений после ratio test |
|---|---|---|
| 1 | 100–150 | 80–130 → стабильно |
| 4 | 40–60 | 30–50 → уверенно |
| 9 | 20–35 | 10–20 → ниже порога |

После извлечения FLANN ищет совпадения этих 500 точек с сохранёнными дескрипторами каждого объекта по очереди. Один и тот же набор точек кадра — один раз для chip-5, один раз для chip-25, и т.д. Точки не «принадлежат» конкретной фишке — они общий ресурс.

**Что с этим делать:**
- Увеличить `SIFT_FEATURES` до 1000–1500: больше точек на весь кадр → больше на каждую фишку. Цена: ~30–50% рост `extract` времени.
- Фокусированная детекция: при ходе конкретного игрока передавать в детектор только его фишку. Все 500 (или 1500) точек работают на один объект → 200–400 совпадений → надёжная детекция даже для слабых объектов.
